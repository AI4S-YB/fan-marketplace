1. 安装 Claude Code

Claude Code 是运行 AI Skill 的编程 agent 环境。需要 Node.js 18+。我们提供了一键安装脚本：

macOS / Linux · 国内用户

$ curl -fsSL https://cdn.jsdelivr.net/gh/AI4S-YB/fan-marketplace@main/install_claude_code.sh | bash

macOS / Linux · International

$ curl -fsSL https://raw.githubusercontent.com/AI4S-YB/fan-marketplace/main/install_claude_code.sh | bash

该脚本会自动检测网络环境（国内/国外），选择最快的镜像源安装 nvm、Node.js 和 Claude Code CLI。

手动安装

1. 检查 Node.js——Claude Code 需要 Node.js 18 或更高版本：

$ node --version
# 如输出 v18.x.x / v20.x.x / v22.x.x 即满足要求
# 如提示 command not found，说明需要先安装 Node.js

2. 如未安装 Node.js，推荐通过 nvm 安装：

# macOS / Linux
$ curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash
# 重启终端，然后安装 Node.js LTS：
$ nvm install --lts
$ nvm use --lts
$ node --version                     # 确认版本 >= 18

nvm 安装如遇网络问题，可参考 nvm 国内镜像。

3. 确认 Node.js 版本后，通过 npm 安装 Claude Code：

$ npm install -g @anthropic-ai/claude-code
$ claude --version                    # 确认安装成功

Windows

Windows 用户推荐安装 WSL2（Windows Subsystem for Linux），在 Linux 子系统中运行 Claude Code：

# 以管理员身份打开 PowerShell，执行：
$ wsl --install
# 重启后进入 Ubuntu 终端，然后按上方 macOS/Linux 步骤操作

WSL2 安装指南：Microsoft WSL 官方文档

安装完成后，终端输入 claude 启动 Claude Code。首次使用需要 Anthropic API Key 或登录 Claude 账号。

2. 配置 Settings — 接入大模型

Claude Code 的配置文件位于 ~/.claude/settings.json。如果该文件不存在，手动创建即可：

$ mkdir -p ~/.claude
$ touch ~/.claude/settings.json

下面给出完整的配置示例。将 ANTHROPIC_AUTH_TOKEN 替换为你自己的 API Key。

接入 DeepSeek（推荐）

DeepSeek 国内可用、性价比高，适合日常使用。获取 API Key：platform.deepseek.com

{
  "env": {
    "ANTHROPIC_AUTH_TOKEN": "你的-deepseek-api-key",
    "ANTHROPIC_BASE_URL": "https://api.deepseek.com/anthropic",
    "ANTHROPIC_DEFAULT_HAIKU_MODEL": "deepseek-v4-flash",
    "ANTHROPIC_DEFAULT_SONNET_MODEL": "deepseek-v4-pro",
    "ANTHROPIC_DEFAULT_OPUS_MODEL": "deepseek-v4-pro",
    "ANTHROPIC_MODEL": "deepseek-v4-pro[1m]",
    "API_TIMEOUT_MS": "3000000",
    "CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC": "1",
    "CLAUDE_CODE_DISABLE_NONSTREAMING_FALLBACK": "1",
    "CLAUDE_CODE_EFFORT_LEVEL": "max",
    "CLAUDE_CODE_SUBAGENT_MODEL": "deepseek-v4-pro"
  },
  "model": "opus[1m]",
  "permissions": {
    "allow": [
      "Read(*)",
      "Write(*)",
      "Edit(*)",
      "Glob(*)",
      "Grep(*)",
      "Bash(*)"
    ]
  }
}

权限说明：以上仅开放文件读写编辑、代码搜索和终端命令。如需 Notebook 支持，可自行添加 "NotebookRead(*)" 和 "NotebookEdit(*)"。

接入智谱 GLM-4

智谱 GLM-4 是合规场景优选。将上面配置中的 ANTHROPIC_BASE_URL 改为：

"ANTHROPIC_BASE_URL": "https://open.bigmodel.cn/api/paas/v4"

模型名和 API Key 也对应替换。获取 API Key：open.bigmodel.cn

3. 安装 fan-marketplace

安装前，确认 nvm 已正确加载：

$ nvm --version
# 正常输出：0.40.3（或类似版本号）
# 如提示 command not found，说明 nvm 未安装或未加载

未安装 nvm，请回到 第 1 步 用一键脚本安装。

已安装但未加载（安装后还没新开终端），执行：

$ source ~/.zshrc          # macOS 默认
$ source ~/.bashrc         # Linux 默认

确认 nvm --version 有输出后，继续安装：

# 方式一：npm registry（推荐）
$ npm install -g fan-marketplace
# 方式二：GitHub 直接安装（npm 不可用时备选）
$ npm install -g https://github.com/AI4S-YB/fan-marketplace.git
$ fan --help

未用 nvm（系统包管理器安装的 Node.js）：全局安装需要 sudo。建议先配 npm prefix 避免权限问题：

$ mkdir -p ~/.npm-global
$ npm config set prefix ~/.npm-global
$ echo 'export PATH="$HOME/.npm-global/bin:$PATH"' >> ~/.zshrc
$ source ~/.zshrc

安装完成后，用 fan install 安装的 Skill 会自动以软链接同步到 ~/.claude/skills/，Claude Code 自动发现，无需额外配置。

4. 发现与安装 Skill

安装 fan-marketplace 后，即可浏览和安装 Skill：

$ fan search "plant"                     # 搜索 Skill
$ fan info fan-skill                    # 查看 Skill 详情
$ fan install fan-skill                 # 安装 Skill（自动处理依赖）
$ fan list                              # 列出已安装的 Skill
$ fan capabilities                      # 显示所有可用能力
$ fan which consult-analysis            # 查找某个能力由谁提供
$ fan invoke fan-skill consult-analysis # 调用某个能力

安装后重启 Claude Code（或开启新对话），Skill 即自动出现在可用技能列表中。

5. 以 Claude Code 插件方式安装（非官方）

如果已在使用 Claude Code，可在对话中直接注册 FAN Marketplace：

# 在 Claude Code 对话中执行：
/plugin marketplace add AI4S-YB/fan-marketplace
/plugin install fan-skill@fan-marketplace

注意：此方式需要 Claude Code 能访问 GitHub。如果网络受限，推荐使用上方 fan install 命令（通过 jsDelivr CDN 获取 registry）。两种安装方式互通，fan list 均可见。

用 AI 创建你的 Skill

将下面的提示词粘贴到 Claude Code 对话框，AI 会根据下方的模板为你生成完整的 Skill 仓库：

# 在 Claude Code 中执行（替换 "..." 为你的 Skill 信息）：
帮我创建一个 FAN Marketplace Skill。

名称：...
用途：...
作者（GitHub 用户名）：...
提供什么能力（一句话）：...

请按照本页「skill.yaml 模板」和「SKILL.md 模板」的格式创建文件。
完成后初始化 git 仓库并提交。

把上面的提示词整段复制到 Claude Code。AI 会参照下方模板生成文件，你只需要提供基本信息。

skill.yaml 模板（逐字段说明）

将下面的内容保存为 skill.yaml，替换 ... 部分：

# 唯一标识符，全小写字母+数字+连字符，不可与已有 Skill 重复
name: my-skill
# 语义化版本号，格式 x.y.z。首次发布建议 0.1.0
version: 0.1.0
# 在 Marketplace 页面上显示的名称
display_name: "My Skill"
# 一句话描述 Skill 做什么。会显示在搜索结果和卡片中
description: "一句话描述这个 Skill 的功能"
# GitHub 用户名或组织名
author: your-github-handle
# 开源许可证，推荐 MIT
license: MIT
# Skill 仓库的 GitHub 地址
homepage: https://github.com/YOU/my-skill
# 运行配置（通常不需要改）
runtime:
  type: skill                    # skill | tool | data
  min_fan_version: "0.1.0"       # 要求的最低 fan-cli 版本
# 此 Skill 对外暴露的能力列表
provides:
  - id: my-capability            # 能力唯一 ID
    description: "这个能力做什么"  # 能力描述
    # run: echo hello          # 可选：CLI 调用的命令。不加则默认输出 SKILL.md
# 此 Skill 依赖的其他能力 ID 列表。如不依赖则留空
requires: []
# 搜索关键词，用于 Marketplace 搜索和分类。建议 5-10 个
keywords: [bioinformatics, genomics]
# 分类标签：core（核心） | community（社区贡献）
category: community
# 可选：此 Skill 需要的外部系统工具（非 Skill 依赖）
# external_deps:
#   - name: curl
#     install_hint: "apt install curl"
#     required: true

SKILL.md 模板

SKILL.md 是加载到 AI 上下文中的指令文件。将下面的内容保存为 SKILL.md：

# YAML frontmatter（Claude Code 必需，不要删除）
---
name: my-skill
description: 当用户需要 [做什么] 时使用此 Skill
---

# 以下为 AI 指令正文（自由格式 Markdown）

## 你的角色
你是一个专门负责 [领域] 的 AI 助手。

## 工作流程
当用户提出 [需求] 时，按以下步骤执行：

1. 首先，[步骤一]
2. 然后，[步骤二]
3. 最后，[步骤三]

## 约束规则
- 始终 [规则]
- 严禁 [规则]
- 如果遇到 [情况]，请 [处理方式]

## 示例
用户：「...」
你的回复：「...」

frontmatter 中的 description 决定 Claude Code 何时自动激活此 Skill。正文写清楚 AI 的行为规范，越具体越好。

注册到 FAN Marketplace

Skill 仓库推送后，只需在 registry-list.json 中添加你的仓库地址，提交 Pull Request：

# 在 fan-marketplace 仓库的 registry-list.json 末尾加一行：
"https://github.com/YOU/my-skill"

就这么简单！CI 会自动从你仓库的 skill.yaml 读取所有信息，生成 index.json。你不需要手动写任何 JSON。

流程：Fork fan-marketplace → 编辑 registry-list.json → 加一行仓库地址 → 提 PR → 维护者审核合并。

关于 FAN Marketplace

FAN 是一个面向农业科学与生物育种的 AI 技能开放生态。我们的目标是通过 AI 工具连接生物学问题与计算分析，让每一位研究者都能获得专业的生物信息学支持。

旗舰项目 fan-skill 覆盖从 GWAS、基因组选择到转录组学、系统发育等多个分析领域。每个领域结合专家知识（C 层决策规则）与 AI 推理（B 层智能），引导研究者从科学问题到可发表的结果。

运行机制

Skill 是 AI agent 插件。安装后，Claude Code 获得该领域的专业知识、分析流程和工具集成。开启对话，AI 即刻具备领域专长。

Skill 之间可以互相调用。fan-skill 负责分析规划，当需要 NCBI 数据时，自动发现并调用 ncbi-downloader skill。生态随每个新 skill 不断成长。
