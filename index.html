1. 安装 Claude Code

Claude Code 是运行 AI Skill 的编程 agent 环境。需要 Node.js 18+。我们提供了一键安装脚本：

macOS / Linux · 国内用户

$ curl -fsSL https://cdn.jsdelivr.net/gh/AI4S-YB/fan-marketplace@main/install_claude_code.sh | bash

macOS / Linux · International

$ curl -fsSL https://raw.githubusercontent.com/AI4S-YB/fan-marketplace/main/install_claude_code.sh | bash

该脚本会自动检测网络环境（国内/国外），选择最快的镜像源安装 nvm、Node.js 和 Claude Code CLI。

或手动安装：

$ npm install -g @anthropic-ai/claude-code
$ claude --version

Windows

Windows 用户推荐安装 WSL2（Windows Subsystem for Linux），在 Linux 子系统中运行 Claude Code：

# 以管理员身份打开 PowerShell，执行：
$ wsl --install
# 重启后进入 Ubuntu 终端，然后按上方 macOS/Linux 步骤操作

WSL2 安装指南：Microsoft WSL 官方文档

安装完成后，终端输入 claude 启动 Claude Code。首次使用需要 Anthropic API Key 或登录 Claude 账号。

2. 配置 Settings — 接入第三方模型

Claude Code 默认使用 Anthropic 官方模型。如果你需要接入 DeepSeek、GLM 等第三方模型以降低成本或满足合规要求，编辑 ~/.claude/settings.json：

接入 DeepSeek 模型

{
  "model": "deepseek/deepseek-chat",
  "apiKey": "sk-your-deepseek-api-key",
  "baseURL": "https://api.deepseek.com/v1"
}

DeepSeek 是国内可用、性价比高的大模型。获取 API Key：platform.deepseek.com

接入智谱 GLM 模型

{
  "model": "zhipuai/glm-4",
  "apiKey": "your-zhipu-api-key",
  "baseURL": "https://open.bigmodel.cn/api/paas/v4"
}

智谱 GLM-4 是国内合规场景的首选。获取 API Key：open.bigmodel.cn

3. 安装 fan-cli

fan 命令行工具用于搜索、安装和管理 FAN Marketplace 上的 Skill。

$ npm install -g fan-cli
$ fan --help

fan-cli 负责连接 marketplace registry 与本地 AI 编程环境。安装的 Skill 自动同步到 Claude Code。

4. 发现与安装 Skill

$ fan search "plant"               // 搜索 Skill
$ fan info fan-skill              // 查看 Skill 详情
$ fan install fan-skill           // 安装 Skill
$ fan list                        // 列出已安装的 Skill
$ fan capabilities                // 显示所有可用能力
$ fan which consult-analysis      // 查找能力提供者
$ fan invoke fan-skill consult-analysis  // 调用某个能力

安装后重启 Claude Code（或开启新对话），Skill 会自动出现在可用技能列表中。

5. 以 Claude Code 插件方式安装（非官方）

如果已在使用 Claude Code，可直接通过插件市场注册 FAN：

// 在 Claude Code 对话中执行：
/plugin marketplace add AI4S-YB/fan-marketplace
/plugin install fan-skill@fan-marketplace

此方式安装的 Skill 同样由 fan-cli 管理依赖同步。两种安装方式互通，fan list 均可见。

创建你自己的 Skill

每个 Skill 是一个包含 skill.yaml 清单文件和 SKILL.md AI 上下文文件的 git 仓库。

Step 1: 编写 skill.yaml

清单文件声明 Skill 的标识、能力和依赖。

name: my-skill
version: 0.1.0
display_name: "My Skill"
description: "What this skill does in one sentence"
author: your-github-handle
license: MIT
homepage: https://github.com/YOU/my-skill
runtime:
  type: skill
  min_fan_version: "0.1.0"
provides:
  - id: my-capability
    description: What this capability does
    run: echo hello
requires: []

关键字段：name 和 version 必填。provides 列出 Skill 能做什么，每个 capability 可配置 run 命令供 CLI 调用。requires 列出此 Skill 依赖的能力。

Step 2: 编写 SKILL.md

SKILL.md 是加载到 AI agent 上下文的指令文件。编写清晰、直接的指令给 AI。

---
name: my-skill
description: What this skill does for AI agents
---

## How You Work
You are an AI assistant specialized in [domain]. When the user asks about [topic]:

1. First, [step one]
2. Then, [step two]
3. Finally, [step three]

## Constraints
- Always [rule]
- Never [rule]

YAML frontmatter（--- 标记之间）是 Claude Code 必需的。正文为自由 Markdown 格式。

Step 3: 文件结构

my-skill/
├── skill.yaml         // Marketplace 清单文件
├── SKILL.md           // AI agent 上下文
└── // 其他文件 — 脚本、参考文档、数据

Step 4: 注册到 FAN Marketplace

Fork fan-marketplace，在 index.json 中添加你的 Skill 条目，提交 PR：

{
  "id": "my-skill",
  "display_name": "My Skill",
  "description": "What this skill does",
  "author": "your-handle",
  "version": "0.1.0",
  "license": "MIT",
  "homepage": "https://github.com/YOU/my-skill",
  "distribution": {
    "type": "git",
    "url": "https://github.com/YOU/my-skill.git"
  },
  "provides": ["my-capability"],
  "requires": [],
  "keywords": ["your-keywords"],
  "category": "community"
}

合并后，所有人都可以通过 fan install my-skill 安装你的 Skill。

关于 FAN

FAN 是一个面向农业科学与生物育种的 AI 技能开放生态。我们的目标是通过 AI 工具连接生物学问题与计算分析，让每一位研究者都能获得专业的生物信息学支持。

旗舰项目 fan-skill 覆盖 29 个分析领域 —— 从 GWAS、基因组选择到转录组学、系统发育。每个领域结合专家知识（C 层决策规则）与 AI 推理（B 层智能），引导研究者从科学问题到可发表的结果。

运行机制

Skill 是 AI agent 插件。安装后，Claude Code 获得该领域的专业知识、分析流程和工具集成。开启对话，AI 即刻具备领域专长。

Skill 之间可以互相调用。fan-skill 负责分析规划，当需要 NCBI 数据时，自动发现并调用 ncbi-downloader skill。生态随每个新 skill 不断成长。
